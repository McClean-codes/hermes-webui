Setup guide

Hermes, on every device

Hermes WebUI is your personal Hermes backend — run it once, then reach it from a clean, native client on the web, your desktop, and your phone. A Claude- or Codex-style experience everywhere, no terminal required.

🚀

How it fits together. Hermes WebUI is a complete Hermes backend API — one server that holds your conversations, memory, scheduled jobs, workspace, and the full power of the Hermes Agent. Every app below is a client that connects to that one server, so your context follows you from laptop to phone to browser. Set the server up once; add as many clients as you like.

1 Run the server

Stand up Hermes WebUI

On any machine you control — macOS, Linux, or Windows via WSL2 — with Python 3.11+ and git. The bootstrap finds or installs Hermes Agent for you, sets up the environment, and walks you through first-run onboarding.

bash — clone & start
git clone https://github.com/nesquena/hermes-webui.git
cd hermes-webui
python3 bootstrap.py

By default the server listens on 127.0.0.1:8787 — local-only until you expose it in Step 2. Open that URL in any browser and you already have the full Web UI. To relaunch later, run ./start.sh from the repo.

Need Hermes Agent first?

Hermes WebUI runs on top of the Hermes Agent. If it's not already installed, bootstrap.py offers to install it — or set it up directly:

bash — install Hermes Agent
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
source ~/.bashrc      # reload your shell
hermes model          # choose your LLM provider

Full agent walkthrough: the Hermes installation guide →

Set a password before you expose it

Auth is off by default on localhost. Before putting the server on a network (Step 2), set a password — either in the Web UI's Settings panel, or inline:

bash — start with a password
HERMES_WEBUI_PASSWORD=your-secret ./start.sh

Verify it's up

bash — health check
curl http://127.0.0.1:8787/health
🎉 That's the whole server. If you only ever use Hermes in a browser on the same machine, you're done — open http://127.0.0.1:8787 and start chatting. Want it on your phone or other devices? Continue below.
2 Make it reachable

Expose your server (for remote & mobile clients)

To reach the server from another device — your phone, a second laptop — give it a URL. Pick one path; both end with an address you paste into your client app. Skip this if you only use the browser on the same machine.

🌩️ Cloudflare Tunnel Public HTTPS

Cloudflare Tunnel dials out from your machine, so nothing is port-forwarded and TLS terminates at the edge — real HTTPS, anywhere.

bash — instant tunnel
brew install cloudflared
cloudflared tunnel --url http://localhost:8787

Prints a trycloudflare.com URL. Your password is required here — the hostname is public.

🔒 Tailscale Private

Tailscale puts your server and devices on one private WireGuard network — nothing is exposed to the public internet. Install it on both, sign into the same tailnet, then:

bash — bind to the tailnet
HERMES_WEBUI_HOST=0.0.0.0 HERMES_WEBUI_PASSWORD=your-secret ./start.sh
tailscale ip -4   # your server's 100.x address

Your URL is http://<tailscale-ip>:8787. Plain HTTP is fine — WireGuard encrypts the wire.

Troubleshooting

An app can't reach the server

Work outward. On the server, check curl http://127.0.0.1:8787/health. If that works, open your public/Tailscale URL in another device's browser. Whichever hop fails is the one to fix.

The Cloudflare tunnel URL stopped working

Quick tunnels (trycloudflare.com) get a new URL every time cloudflared restarts. For a stable hostname, create a named tunnel on a domain you've added to Cloudflare, and keep cloudflared running.

The Tailscale address doesn't respond

Confirm both devices are signed into the same tailnet and show as connected. Then make sure the server was started with HERMES_WEBUI_HOST=0.0.0.0 — the default localhost binding isn't reachable over the tailnet.

Sign-in keeps failing

The password in your app must match HERMES_WEBUI_PASSWORD on the server (or the one set in Settings). An inline variable only applies to that launch — persist it in the server's .env for daemon restarts.

Port 8787 is already taken

Start on another port with HERMES_WEBUI_PORT=8788 ./start.sh and use that port in your client URL and tunnel command.

Profile switching is disabled / only one profile shows

That's isolated-profile mode. On a normal single-user install it should be off by default — update to the latest WebUI if you see it unexpectedly.
